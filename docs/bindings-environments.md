# TypeScript Bindings: Multi-Environment & External ABI Support

Generated by `glassbox generate-bindings`.

---

## Runtime Targets

The binding generator supports three runtime targets, selected with `--runtime`:

| Target | Flag | Description |
|--------|------|-------------|
| **Node.js** | `--runtime node` (default) | Uses `child_process.spawn` for the Glassbox simulator. Lets the Stellar SDK use its built-in Node transport. |
| **Browser** | `--runtime browser` | Uses the global `fetch` API. No `child_process`, `fs`, or other Node-only imports. Safe to bundle with webpack/Vite/esbuild. |
| **Universal** | `--runtime universal` | Emits runtime environment-detection code. Works in Node.js, browser, and Electron renderer processes without separate builds. |

### Node.js (default)

```bash
glassbox generate-bindings contract.wasm --output ./src/bindings
# or explicitly:
glassbox generate-bindings contract.wasm --runtime node --output ./src/bindings
```

```typescript
import { MyContractClient } from './src/bindings';

const client = new MyContractClient({
  contractId: 'CDLZFC3...',
  network: 'testnet',
});
```

### Browser

```bash
glassbox generate-bindings contract.wasm --runtime browser --output ./src/bindings
```

The generated `package.json` includes a `"browser"` field that excludes Node-only
modules from bundlers:

```json
{
  "browser": {
    "child_process": false,
    "fs": false,
    "path": false
  }
}
```

The `ErstSimulator` uses an HTTP endpoint instead of spawning a local process:

```typescript
import { ErstSimulator } from './src/bindings/Glassbox-integration';

const sim = new ErstSimulator({
  network: 'testnet',
  rpcUrl: 'https://soroban-testnet.stellar.org',
  // Optionally override fetch (e.g. to add auth headers):
  fetch: (url, init) => window.fetch(url, init),
});
```

### Universal (Node + Browser + Electron)

```bash
glassbox generate-bindings contract.wasm --runtime universal --output ./src/bindings
```

The generated code detects the runtime at startup:

- **Node 18+** / **Electron main process**: uses `globalThis.fetch` (built-in).
- **Node < 18**: falls back to `require('node-fetch')` (install separately).
- **Browser** / **Electron renderer**: uses `globalThis.fetch`.

The generated `package.json` includes conditional exports:

```json
{
  "exports": {
    ".": {
      "browser": "./index.browser.js",
      "default": "./index.js"
    }
  }
}
```

## Custom Soroban RPC Provider

All runtime targets support injecting a custom RPC provider via the `provider`
field in `ClientConfig`. This lets you:

- Point at a private or local RPC node.
- Supply a custom network passphrase (e.g. for private networks).
- Wrap `fetch` to add authentication headers, logging, or retry logic.

```typescript
import { MyContractClient, SorobanProvider } from './src/bindings';

const myProvider: SorobanProvider = {
  rpcUrl: 'https://my-private-rpc.example.com',
  networkPassphrase: 'My Private Network ; January 2026',
  fetch: async (url, init) => {
    const resp = await fetch(url, {
      ...init,
      headers: {
        ...init?.headers,
        Authorization: `Bearer ${process.env.RPC_TOKEN}`,
      },
    });
    return resp;
  },
};

const client = new MyContractClient({
  contractId: 'CDLZFC3...',
  network: 'mainnet',   // used for network passphrase fallback only
  provider: myProvider, // overrides rpcUrl and networkPassphrase
});
```

When `provider` is set:
- `provider.rpcUrl` takes precedence over `rpcUrl` and the built-in network defaults.
- `provider.networkPassphrase` (if set) overrides the SDK's built-in passphrase for the network.
- `provider.fetch` (if set) is passed to the Stellar SDK's `SorobanRpc.Server`.

## External ABI / Spec Files

By default, `generate-bindings` extracts the contract spec from the WASM binary's
`contractspecv0` custom section. You can skip WASM entirely by supplying an
external ABI file with `--spec-file`.

### Supported Formats

| Format | Flag | Description |
|--------|------|-------------|
| **JSON** | `--spec-format json` | Glassbox canonical JSON ABI (produced by `glassbox abi --json`). |
| **XDR** | `--spec-format xdr` | Raw XDR-encoded `ScSpecEntry` bytes (same binary format as the WASM section). |
| *(auto)* | *(omit flag)* | Auto-detected: files starting with `{` are treated as JSON, everything else as XDR. |

### Generating a JSON ABI from a contract

```bash
# Export the ABI from a deployed contract:
glassbox abi --contract-id CDLZFC3... --network testnet --json > contract-abi.json

# Generate bindings from the exported ABI:
glassbox generate-bindings \
  --spec-file contract-abi.json \
  --package my-contract \
  --output ./src/bindings
```

### Generating bindings from an XDR spec file

```bash
# Extract the raw spec section from a WASM file:
glassbox abi --wasm contract.wasm --xdr > contract.xdr

# Generate bindings from the XDR spec:
glassbox generate-bindings \
  --spec-file contract.xdr \
  --spec-format xdr \
  --package my-contract \
  --output ./src/bindings
```

### JSON ABI format

The JSON format matches the output of `glassbox abi --json`:

```json
{
  "functions": [
    {
      "name": "transfer",
      "doc": "Transfer tokens from one account to another.",
      "inputs": [
        { "name": "from",   "type": "Address" },
        { "name": "to",     "type": "Address" },
        { "name": "amount", "type": "U128"    }
      ],
      "outputs": ["Void"]
    }
  ],
  "structs": [
    {
      "name": "TokenInfo",
      "fields": [
        { "name": "name",     "type": "String" },
        { "name": "decimals", "type": "U32"    }
      ]
    }
  ],
  "enums": [
    {
      "name": "Status",
      "cases": [
        { "name": "Active",   "value": 0 },
        { "name": "Inactive", "value": 1 }
      ]
    }
  ],
  "error_enums": [
    {
      "name": "TokenError",
      "cases": [
        { "name": "InsufficientBalance", "value": 1 },
        { "name": "Unauthorized",        "value": 2 }
      ]
    }
  ]
}
```

Supported type strings in the JSON format:

| Soroban Type | JSON string |
|---|---|
| `Bool` | `"Bool"` |
| `U32` / `I32` | `"U32"` / `"I32"` |
| `U64` / `I64` | `"U64"` / `"I64"` |
| `U128` / `I128` | `"U128"` / `"I128"` |
| `U256` / `I256` | `"U256"` / `"I256"` |
| `String` | `"String"` |
| `Symbol` | `"Symbol"` |
| `Address` | `"Address"` |
| `Bytes` | `"Bytes"` |
| `BytesN(N)` | `"BytesN(32)"` |
| `Option<T>` | `"Option<U64>"` |
| `Vec<T>` | `"Vec<Address>"` |
| `Map<K,V>` | `"Map<String, U128>"` |
| `Result<T,E>` | `"Result<U32, Error>"` |
| `(T1, T2)` | `"(U32, String)"` |
| Custom UDT | `"MyStruct"` |

## CLI Reference

```
glassbox generate-bindings [wasm-file] [flags]

Spec sources (mutually exclusive):
  <wasm-file>              Extract spec from a compiled WASM binary
  --spec-file <path>       Load spec from a JSON or XDR ABI file

Flags:
  -o, --output <dir>       Output directory (default: current directory)
  -p, --package <name>     Package name (default: input filename without extension)
      --contract-id <id>   Contract ID embedded in generated README
  -n, --network <net>      Stellar network: testnet (default), mainnet, futurenet
      --runtime <target>   Runtime target: node (default), browser, universal
      --spec-file <path>   External ABI/spec file (JSON or XDR)
      --spec-format <fmt>  Spec file format: json, xdr (auto-detected when omitted)
      --debug-metadata     Emit ABI metadata objects and withDebugMetadata() wrappers
      --wasm-source <path> Source path hint embedded in debug metadata
```
